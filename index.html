论文一、

孪生网络共用权值,身体(特征提取部分)相同,头(输出)不同.将一对图片组(两张)传给孪生网络,在网络的倒数第二层使用L1距离将两个网络的输出进行融合,并度量两者的相似性,网络的最后一层使用全连接(说是使用了注意力机制)并用sigmoid输出得到相似性(1同类0不同类),训练的时候喂数据的方法是一组同类和一组异类交替输入。
网络结构:

小样本学习原因:并不是学习出分辨类别的能力,只是学习出衡量相似性的能力
应用场景:可用于相似性度量,也可用于分类任务,将待分类样本与训练集数据比对,进行投票决定类别(个人观点,必须实验试试)
code: https://github.com/kevinzakka/one-shot-siamese

论文二、

Matching Networks=记忆力LSTM+CNN+attention机制,将query图片和support set有效embedding,使用余弦距离衡量相似性,模型具体操作:
对于support set(func G): 使用一个简单的CNN将图片进行编码后,将每个图片看作一个单词Xt,输入双向LSTM进行二次embedding,ht即为新的embedding
对于query image(func F): 使用CNN将图片进行编码后,使用一个K个LSTM cell进行编码
每个LSTM cell的输入:

  1. Xt=图片的CNN_embedding
  2. ht-1(new)=ht-1(old)+CNN_embedding+attetions_map * support_embedding
  3. attetions_map = softmax(torch.mm(ht-1(old)+CNN_embedding, support_embedding))

最后的一个cell的输出hK-1即为query image的embedding

一、相关概念

  1. hash编码: 将图片编码为0,1串,使用神经网络先生成real-value编码然后取阈值将real-value编码转换为hash编码
  2. hash编码的三个必须要保证的限制: similarity preserving; coding balance; quantization errors;
    similarity preserving即图片在语义空间的相似性在哈希空间也要保持,coding balance即编码bits之间相互独立, quantization errors即real-value编码转换到hash编码的损失要尽量小
  3. Hamming distance: 衡量两个字符串的相似性,将其中一个变为另外一个所需要作的最小替换次数

二、model

第一部分:特征提取器

主干特征提取网络使用Alexnet进行特征提取,使用passthrough进行多尺度特征图聚合,使用注意力机制(attention branch network (ABN)) 生成attention map 根据每个类的特点去突出 the discriminative regions
需要学习一下:《Attention branch network: Learning of attention mechanism for visual explanation》

第二部分:hash编码器

使用对抗正则模型迫使生成的value code逼近离散均匀分布

三、主要思想

end to end网络, 模型同时进行特征提取和编码, 具体是特征提取后进行real-value编码再将real-value编码转换为hash编码, 特点是使用了特征融合和注意力机制进行特征提取,使用对抗正则网络迫使生成的数据逼近于一个指定的分布

论文积累5

发表于 2020-06-10 | 分类于 Paper

A Discriminative Distillation Network for Cross-Source Remote Sensing Image Retrieval

目录:

  1. 相关概念
  2. model
  3. 主要思想
  4. 理论知识点
  5. 实验仿真、实验结果分析、github代码

一、相关概念

  • Knowledge Distillation(知识蒸馏): 将知识从复杂笨重的网络迁移给较轻便的网络,以压缩/训练模型。本文中使用知识蒸馏并不是为了压缩模型,而是根据不同模态遥感图像低层特征不同,高层语义近似的特点,把f网络学到的高层特征给G网络以加快G网络收敛,同时促使后一模态图像的特征逼近于前一模态的特征,使二者在同一特征空间。在该论文中使用logits计算MSEloss,而不是使用softmax后计算crossentropyloss应该是由于该网络并不是一个用于的分类网络,网络学到的是图片的特征向量而不是一组概率值。
    参考博客:https://zhuanlan.zhihu.com/p/90049906

  • attention mechanism: 注意力分配,重要的值得注意的东西权值高。

  • multitask learning:😔

  • cross-model retrieval:query图像和数据库图像不同源,为不同类型遥感图像

  • Metric learning: 相似度学习,学习一个度量相似度的距离函数:相似的目标离得近,不相似的离得远

  • person re-identification:行人重识别,行人重识别问题通常看做是一个图片检索问题，具体地讲就是在不同摄像头中进行行人匹配，给出一个行人目标，判断其他摄像头是否也检测到这个行人。

    二、Model


使用两个架构完全相同的特征提取网络,训练方法是:先训练一个特征提取网络,然后将学到的参数传给第二个网络,固定第二个网络的高层参数,只训练低层部分,使得学到的特征和第一个网络的特征在同一个特征空间

三、主要思想

使用知识蒸馏思想进行跨模态图像检索,使用知识蒸馏网络,使不同源图片训练出来的特征在同一个特征空间,使用triplet loss提高类间分散性和类内致密性,使用Center loss减小类内距离

四、理论知识点

  1. t-SNE(t分布随机邻域嵌入)

  2. triplet loss

    对于batch中的每一个图片,在batch中选择一个同类的图片作为positive anchor,选择一个不同类图片作为negetive anchor

  1. pytorch 中的 CrossEntropyLoss详解
    ⚠️:使用CrossEntropyLoss,softmax回归后的输出size=(batchsize, num_class)后不必进行softmax,且计算得到的损失是进行了(求和/batchsize)操作的。
    https://www.pianshen.com/article/4916145350/
    https://pytorch.org/docs/stable/nn.html?highlight=cross#torch.nn.CrossEntropyLoss

五、实验结果分析

效果糟糕,检索100张图片正确率80%左右,主要是代码实现问题,我的损失函数实现及数据读取方式需要改进, pytorch编程问题还是蛮大的

  2. 什么是过拟合? 解决过拟合（over fitting）的途径有哪些？并对每种途径举例说明。

    1. 使用简单模型
      (1) 决策树剪枝
      复杂的决策树(结点多,层数多)对于训练集可以达到很高的分类正确率,但是对于测试集往往出现过拟合, 剪枝即是将已生成的树进行化简,将复杂模型简单化。
      (2) SVM中引入松弛因子
      支持向量机分为线性可分支持向量机, 线性不可分支持向量机, 非线性支持向量机,线性可分支持向量机只能分类线性可分数据, 但是引入松弛因子后svm可以分类线性不可分数据, 由于噪声点的存在, 尤其是位于超平面附近的噪声点若成为支持向量会极大的影响超平面的位置而影响分类性能, 因此引入松弛因子以后, 模型容许部分样本被错分或着位于间隔边界内, 使得模型不会过分苛求对于训练集的完全可分, 避免过拟合。

    2. 数据清理和修剪
      数据清理和修剪是对训练数据集里label明显错误的样本进行修正，或者对错误的样本看成是噪声，进行剔除, 其关键在于如何准确寻找label错误的点或者是noise的点，而且如果这些点相比训练样本N很小的话，这种处理效果不太明显。
      (1) 在模式识别课程中的曾讲解了基于KNN的改进版本剪辑近邻法, 该方法在减少数据量的同时,也降低了错误率, 其主要思想就是错误率主要来自于分类交叠区域,因此去掉两类边界附近的样本可以提高正确率。避免对于训练数据的过度依赖产生的过拟合。
      (2) 在数据挖掘课程中, 其数据预处理阶段, 常使用基于统计分布的3δ原则, 箱线图, 基于距离, 基于密度的离群点检测方法去检测噪点, 并用平均值替代,删除等方法去噪。

    3. data hinting
      data hinting是针对训练样本N不够大的情况，如果没有办法获得更多的训练集，那么data hinting就可以对已知的样本进行简单的处理、变换，从而获得更多的样本。
      (1)在分类问题中，可以对已知的图片进行轻微的平移或者旋转，从而让N丰富起来，达到扩大训练集的目的。
      (2)基于VAE和GAN的生成技术可以生成较为简单的训练数据,从而扩大训练集数量。

    4. 正则化
      在损失函数中引入正则化惩罚项
      L2-norm:损失函数加上λ/2 ‖w‖^2
      L1-norm:损失函数加上λ‖w‖_1
      ElasticNet:结合L1正则和L2正则
      (1) 该方法在机器学习算法中经常使用, 例如SVM的损失函数自带正则项, ridge回归,lasso回归,ElasticNet都是在线性回归基础上加上正则项迫使高位项参数较小从而降低阶数不至于过拟和。
      (2) 在深度学习网络训练中也常对网络中的参数进行正则。

    5. 交叉验证(引用一张经典图片)

      使用交叉验证,不但可以通过验证集去调节超参数, 而且可以通过验证集去评估模型是否过拟合, 模型在验证集上性能不错,在测试集上性能应该也还行。

      以下解释copy自Solong1919的博客: https://www.cnblogs.com/solong1989/p/9415606.html
      第一个作用就是对模型的性能进行评估。当我们通过一次划分样本对模型进行训练和测试时，由于样本划分的偶然性，会导致我们对模型的评估不准确。因此，可以采用交叉验证对模型进行评估（一般采用5折或10折，sklearn默认采用的是3折），以 n 折交叉验证结果的均值，作为模型的性能评估。
      第二个作用就是用来避免过拟合。例如当我们进行10折交叉验证时，训练了10次，得到了10个模型，每个模型的参数也是不同的，那么我们究竟用哪个模型作为我们最终的模型呢？答案是：一个都不用！我们要利用全量数据重新训练出一个最终模型！
      如果你觉得有点迷糊的话不用急，我们一起来梳理一下。我们训练模型是为了什么？是为了确定模型中的参数，例如多元回归模型的系数、神经网络各神经元的权重等。但是，有些模型除了参数还有超参数，例如KNN和KMeans的k值，支持向量机的C值，这些超参数是需要人为设定，没有办法让模型自己学习得到，我们只能通过经验或者多次训练比较，主观设定一个我们认为相对最优的值。而交叉验证的作用，就是让我们进行多次的模型训练比较。例如，我们期望从 [ 1, 10, 100, 1000 ] 这几个值中选择一个最优值，作为支持向量机的C值。那么我们就得分别用这4个值进行10折交叉验证（该过程经历了40次模型的训练测试），把每一次交叉验证结果的均值进行比较，从而选择出一个最优值。最优值选出来了，那么模型的超参数确定了，接下来，利用全量的数据（不要划分训练集测试集，而是用全部的数据）进行模型训练，训练出来的模型才是我们的最终模型。在此过程中可以看出，交叉验证帮助我们确定模型的超参数。这样训练出来的模型，是经过多次综合比较得出的相对最优模型，在一定程度上可以避免过拟合的问题，这就是为什么我们会说交叉验证可以避免过拟合。

特征处理

1. 类别可分性判据

顺便复习一下模式识别

  • 类别可分性判据: 定量衡量特征组对分类任务好坏
  • 类别可分性判据的四大准则: 与错误概率单调; 度量特性(两类之间可分性判据函数值越大,两类差别越大); 特征数增加,其值不减小;
  • 三种可分性判据:
    • 基于距离的可分性判据
      • 总思想:类内距离最小,类间距离最大(类内样本离散度矩阵、类间 样本离散度矩阵)。
    • 基于概率分布的可分性判据
      • 总思想:使用类条件概率分布衡量两类特征相似性, 若两概率分布重叠大,特征越不好两类越不容易区分。
      • 概率函数相似性度量方法:
        • 巴氏距离
        • 切比雪夫距离
        • 散度(尚未理解)
    • 基于熵的可分性判据
      • 总思想:P(y|x)后验概率的熵越小,越可分

2. 特征选择与特征变换

  • 定义
    • 特征选择: 是指从原有的D维特征空间，删去一些特征描述量，从而得到精简后的特征空间。在这个特征空间中，样本由降维后的d维的特征向量描述:x={x1,x2,…xd}，d<D。由于x只是y的一个子集，因此每个分量xi必然能在原特征集中找到其对应的描述量xi=yi。
    • 特征变换: 找到一个映射关系:A:Y→X，使新样本特征描述维数比原维数降低。其中每个分量xi是原特征向量 各分量的函数，即xi=W^T·yi。
  • 目的: 特征降维
  • 方法
    • 变换
      • 有监督方法
        1. LDA(大佬写的真的太好了)
          https://www.cnblogs.com/pinard/p/6244265.html
      • 无监督方法
        1. PCA https://www.cnblogs.com/pinard/p/6239403.html#undefined
    • 选择
      • filter方法:根据独立于分类器的指标J来评价所选择的特征子集S，在所有可能的特征子集中搜索出使得J最大的特征子集作为最优特征子集。 不考虑所使用的学习算法。
      • Wrapper方法:将特征选择和分类器结合在一起, 在分类过程中表现优异的的特征子集会被选中。
      • 嵌入法: 利用正则化的思想，将部分特征属性的权重调整到0，则这个特性相当于就是被舍弃了。就是在损失函数上再加入正则项,不断的利用梯度下降极小化损失函数,调整一些特征的权重,有些权重变为0了则相当于被舍弃了，没被舍弃的相当于被选择出来的向量。
